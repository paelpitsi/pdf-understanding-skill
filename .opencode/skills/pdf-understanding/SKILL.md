# Skill: PDF Understanding

Converts PDF to structured markdown. Uses `pdftoppm` for page rendering and a
subagent with a vision model for content analysis.

## Requirements

- **poppler-utils** (`pdftoppm` command)
  - Windows: `winget install --id=oschwartz10612.Poppler -e`
  - macOS: `brew install poppler`
  - Linux: `apt install poppler-utils` or `pacman -S poppler`
- **OpenCode Zen** provider with API key (for the vision model)
- **DeepSeek** provider (main agent, free)

## Process

0. **Determine scope:**
   - If the user says "all", "all pdfs", "batch" or passes a folder — collect
     PDFs:
     - Project folder: `glob "**/*.pdf"`
     - Specific folder: `glob "<path>/*.pdf"`
   - If the user passes a file path — process a single PDF
   - Run steps 1-7 on each PDF

1. **Check dependencies:** if `pdftoppm` is not found — install via system
   package manager

2. **Setup:** create `output/` directory (if not exists)

3. **Render pages:**
   `pdftoppm -png -r 200 "<pdf_path>" "output/pages/<filename>/page"`
   → each page becomes `page-01.png`, `page-02.png`, etc.

4. **Batching:** split pages into groups of 10

5. **Parallel analysis:** for each group, dispatch a `pdf-vision-agent`
   subagent via `task`:
   - Instruct: "Read files page-NN.png ... page-MM.png and describe each page"

6. **Assembly:** concatenate all subagent results in page order

7. **Save:** write final markdown to `output/<filename>.md`

8. **Report:**
   - Single file: "Done: output/<filename>.md"
   - Batch: summary table — files processed, pages, result, any failures

## Output Format

```markdown
# <Filename>

Source: <path to PDF>
Pages: <total>
Processed: <date>

---
[concatenated subagent results]
---
```

## Important Details

- **Parallelism:** process up to 3 subagents concurrently per batch
- **Pipeline limit:** do not run more than 3 batch pipelines simultaneously.
  If many PDFs — process sequentially to avoid API rate limits
- **Error handling:** if a subagent does not respond — skip its group and
  continue. Report which pages were not processed at the end
- **Noise images:** do not manually filter — the vision model decides what is
  decorative
- **Mermaid:** generate only for diagrams, graphs, and flowcharts — not for
  photos
- **Fallback model:** if MiMo Free is unavailable — use
  `google/gemini-2.5-flash` (via OpenRouter, configure in opencode.json)
- **Cleanup:** after each PDF, delete temporary PNGs
  (`output/pages/<filename>/`) to save disk space
