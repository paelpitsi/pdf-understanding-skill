# Skill: PDF Understanding

Преобразует PDF в структурированный markdown. Использует pdftoppm для
рендеринга страниц и subagent с vision-моделью для анализа.

## Требования

- **poppler-utils** (команда `pdftoppm`)
  - Windows: `winget install --id=oschwartz10612.Poppler -e`
  - macOS: `brew install poppler`
  - Linux: `apt install poppler-utils` или `pacman -S poppler`
- **OpenCode Zen** провайдер с API-ключом (для vision-модели)
- **DeepSeek** провайдер (основной агент, бесплатно)

## Процесс

0. **Определить цель:**
   - Если пользователь сказал "all", "все", "all pdfs", "batch" или
     передал папку — собрать список PDF:
     - Папка проекта: `glob "**/*.pdf"`
     - Конкретная папка: `glob "<путь>/*.pdf"`
   - Если пользователь передал путь к файлу — обработать один PDF
   - На каждом PDF выполнить шаги 1-7

1. **Проверка зависимостей:** если `pdftoppm` не найден — установить через
   пакетный менеджер системы

2. **Подготовка:** создать папку `output/` (если ещё нет)

3. **Рендеринг страниц:**
   `pdftoppm -png -r 200 "<pdf_path>" "output/pages/<имя_файла>/page"`
   → каждая страница станет `page-01.png`, `page-02.png` и т.д.

4. **Батчинг:** разделить страницы на группы по 10

5. **Параллельный анализ:** для каждой группы запустить subagent
   `pdf-vision-agent` через `task`:
   - Передать инструкцию: "Прочитай файлы page-NN.png ... page-MM.png и
     опиши каждую страницу"

6. **Сборка:** склеить результаты всех subagent'ов в порядке страниц

7. **Сохранение:** записать итоговый markdown в `output/<имя_файла>.md`

8. **Отчёт:**
   - Для одного файла: "Готово: output/<имя_файла>.md"
   - Для batch: сводная таблица — какие файлы обработаны, страниц,
     результат, какие не удались

## Формат вывода

```markdown
# <Имя файла>

Источник: <путь к PDF>
Страниц: <всего>
Обработано: <дата>

---
[склеенные результаты от subagent'ов]
---
```

## Важные детали

- **Параллельность:** запускать до 3 subagent'ов одновременно
- **Обработка ошибок:** если subagent не ответил — пропустить группу,
  продолжить. В конце указать какие страницы не обработаны
- **Мусорные изображения:** не фильтровать вручную — vision-модель сама
  решает что декоративное
- **Mermaid:** генерировать только для диаграмм, графов и блок-схем,
  не для фото
- **Fallback модель:** если MiMo Free не работает — использовать
  `google/gemini-2.5-flash` (через OpenRouter, настроить в provider)
- **Batch processing:** не запускать больше 3 подряд pipeline'ов
  одновременно. Если PDF много — обрабатывать последовательно, чтобы
  не упереться в лимиты
- **Очистка:** после каждого PDF удалять временные PNG
  (`output/pages/<имя_файла>/`), чтобы не забивать диск
